Cost Increase Explanation

SOC2 Security Requirements (+$16,200, +3.6 weeks): Implementation of comprehensive security controls, audit trails, and compliance documentation required for SOC2 certification significantly increased development complexity.


Bluetooth Gateway Integration (+$10,800, +2.2 weeks): Adding support for a new Bluetooth gateway device type required additional firmware considerations, new data conduits, and expanded device management capabilities beyond the original scope. The primary complexity increase comes from the need to secure every device on the communications chain. Prior to this change, the project was scoped assuming the end device (sensor) would report directly to the AWS cloud via wifi or cellular. With this change, TLS support, certificates, and keys will be required for the devices and the gateway.

Risk Assessment

Comprehensive risk analysis with mitigation strategies

Major Questions & Technical Challenges

Device & Gateway Emulation

How do we emulate a device and a gateway? Emulating a device is easy, but emulating the entire data path is more complex: device ↔ gateway ↔ AWS

Mitigation: Decide on gateway and device management and trust level up front in phase 0.

Gateway Selection & Firmware

Gateway selection and firmware? We need to complete a trade study and select a gateway. Even if we don't proceed immediately with the gateway, we need to remember that is an additional project.

Mitigation: Define gateway requirements in Phase 0, coordinate with hardware team, and plan firmware development as separate project scope.

TLS Communications Via Gateway

End device communications through the gateway. There are several options: (1) Gateway manages devices as trusted endpoint (2) Both gateway and devices managed (3) Gateway as TLS tunnel

Mitigation: Evaluate security requirements, performance implications, and choose architecture in Phase 0 with client approval.

Technical Risks

Real-time Performance Requirements

1-second latency requirement for live monitoring may be challenging with cloud architecture.

Mitigation: For real-time requirements, consider using a local application with a direct connection to the device.

Hardware Requirements

By nature of defining the cloud architecture, we will need to define the hardware requirements. This is a significant risk as it will require the hardware team to design to the specification defined in this project.

Mitigation: Engage with hardware team early and often. Draw from Conifer's experience with hardware design and development.

Project Assumptions

SOC 2 Type 2 Security

Assumption that all development and testing will be done in a SOC 2 Type 2 compliant environment.

Bluetooth Gateway

Assumption that devices will communicate through a bluetooth gateway to reach the cloud.

Infrastructure Defined Hardware

Assumption that the hardware design, firmware design, and manufacturing process can be defined and driven by this project.
